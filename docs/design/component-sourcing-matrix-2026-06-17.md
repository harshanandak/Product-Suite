# Component Sourcing Matrix

**Date:** 2026-06-17 · **Status:** FINALIZED · **Owner:** Harsha · **Supersedes** the category-level `docs/design/component-source-strategy-2026-06-17.md` by adding the per-component, per-registry mapping. Generated by the `component-sourcing-matrix` research workflow (Shoogle directory of **216 registries** enumerated; recommended registries' licenses verified). Build-ready.

## 0. How to read this

- **source (registry + add command):** the recommended primary source. `@shadcn` items install via the CLI's built-in default registry (`npx shadcn@latest add <item>`) — there is no `@shadcn` namespace lookup in Shoogle. Third-party items use `npx shadcn@latest add @<registry>/<item>`.
- **fit:** judged as *shadcn-based + Tailwind + token-native (inherits our oklch CSS-vars) + React*. `high` = all four out of the box.
- **build effort:** low / medium / high for the suite-specific assembly on top of the source.
- **Engines are decided, not sourced:** TanStack Table v8 + Virtual, dnd-kit, custom Gantt, React Flow, TipTap + y-prosemirror, perfect-freehand, cmdk, Recharts v3, AI SDK v6 + OpenRouter, Clerk (DESIGN.md §10).

## 1. IP Guardrail (binding)

- **(A) MIT / Apache-2.0 / OSI-permissive** → we **VENDOR** the copied source into `packages/ui*` (we own it), editable, with the license/copyright notice preserved. Re-theme to tokens (tokens-not-values law).
- **(B) Paid / "Pro" / non-OSI / license-key-gated / license-unverified** → **INSPIRATION ONLY**. We may re-implement the *design* on our own free shadcn primitives. We never paste their source. This covers all `mixed`-tier Pro catalogs and every `unknown`-tier registry until its license is verified from the homepage/repo.
- **"Free" inside a mixed-tier registry is not automatically vendorable** — confirm per-item SPDX (e.g. Aceternity free=MIT but Pro proprietary; Efferd free-block license unverified).
- **Suite grammar is never imported:** ProposalCard, ProvenanceChip, AssigneePicker, PhasePill/HealthBadge, StatusPill, RunProgress, EmptyState, ErrorState are built in-house.

## 11. Build sequence (given lanes L1 Workboard / L2 Meeting / L3 Agents / L4 Home+Chat)

1. **Phase 0 — Foundations (blocks all):** token layer (@theme, indigo oklch, Geist, 0.5rem, `--sidebar-*`, `--chart-1..5`); vendor @shadcn core + sidebar-07/dashboard-01; build AppShell chrome (WorkspaceSwitcher, BoardDock + Cmd+1..5, per-board SidebarBody contracts, TopBar, Cmd+K palette, ReviewQueueDrawer, UserMenu over Clerk); four-state primitives; resolve the three open Qs (cross-package add output dir, AI SDK v6 transport in Vite+TanStack Router, React 19 peerDep pin).
2. **Phase 1a — L2 Meeting board FIRST:** lowest new-UI risk; reuses shipped ui-meeting MeetingSummaryBlock (PR8) and meeting DB cutover (PR20). Proves the shell + a real board + the meeting→action seam.
3. **Phase 1b — L4 Home + Chat (parallel with Meeting):** Home = shell primitives + ui-charting MetricCard (shipped); Chat vendors @ai-elements (self-contained in ui-chat, PR9). Landing the summonable chat shell + Home digest early gives every later board its 'Ask agent' + review-queue surface.
4. **Phase 1c — L1 Workboard:** highest engine effort (TanStack Table v8 + Virtual, dnd-kit, custom Gantt, React Flow graph) with full action parity across all four lenses. Start after Meeting proves the shell.
5. **Phase 1d — L3 Agents board (after Chat):** reuses the same @ai-elements agent primitives + in-house ProposalCard/RunProgress/StatusPill; centralizes runs/status/cost/artifacts + custom-agent members.
6. **Phase 1e — Canvas board (parallel, engine-isolated):** React Flow + TipTap + perfect-freehand behind ui-canvas adapters; gate on Phase 0 only.
7. **Phase 2 — Cutover:** 302 old routes → /w/[workspace]; retire forbidden vocab (Plan/Docs/Insights/stream/five apps) from UI; verify provenance + single review-queue parity.
8. **Phase 3 — Deferred polish:** motion micro-interactions (re-implement paid/animated registry designs on free primitives per §1); harden AI-dashboard JSON-spec renderer; charts escape hatches (Chart.js/visx) only where needed.
